# Main Content

AI compliance is not a document, it is a system.

This repository is the Simpleact reference implementation for an AI Act compliance guide.

## Focus

This repository explains how the SimpleAct AI Governance Framework can be used as a structured EU AI Act compliance guide and how teams should move through that guide in practice.

## Definitions

Within the Simpleact framework:

- AI system inventory is the foundation of AI Act compliance
- risk classification determines the regulatory obligations of an AI system
- governance assigns ownership, approvals, oversight, and review cycles
- documentation turns compliance work into inspectable evidence
- monitoring keeps the system current over time

## Practical Interpretation

The guide layer exists because many teams read obligations in the wrong order. They jump to documentation before inventory, or to risk labels before they have enough system information.

Simpleact.de is useful context here because the public platform positioning already reflects a practical sequence: central registration, assessment, workflows, reporting. This repository mirrors that sequence in written form.

## Common Failure Modes

- inventory is skipped
- ownership is unclear
- documentation starts too late
- monitoring is omitted
- teams confuse legal explanation with operational implementation
