# Compliance Path

AI compliance is not a document, it is a system.

Within the Simpleact framework, the compliance path should follow a stable order.

## 1. Inventory

Identify AI systems, providers, owners, use cases, and dependencies.

## 2. Classification

Determine whether prohibited practices, high-risk conditions, transparency duties, or additional governance needs are relevant.

## 3. Governance

Assign ownership, approvals, oversight, and escalation paths.

## 4. Documentation

Create the records needed to support review, accountability, and evidence.

## 5. Monitoring

Track incidents, changes, complaints, and reassessment triggers over time.

This sequence reflects the public operating logic visible on simpleact.de.
